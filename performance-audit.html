P1 — Critical

No browser cache headers — 17,640 KiB served fresh on every visit

Lighthouse: Use efficient cache lifetimes — saves 17,640 KiB

The site is hosted on GitHub Pages with no _headers file, so every asset — images, fonts, HTML — is re-downloaded on every page visit. A returning visitor gets zero caching benefit. Adding a _headers file tells GitHub Pages / Cloudflare to cache static assets for up to 1 year.

✅ Fix — Add a _headers file to the repo root
/* X-Content-Type-Options: nosniff X-Frame-Options: DENY /*.png Cache-Control: public, max-age=31536000, immutable /*.webp Cache-Control: public, max-age=31536000, immutable /*.jpg Cache-Control: public, max-age=31536000, immutable /*.svg Cache-Control: public, max-age=31536000, immutable /*.js Cache-Control: public, max-age=31536000, immutable /*.css Cache-Control: public, max-age=31536000, immutable /index.html Cache-Control: public, max-age=0, must-revalidate /*.html Cache-Control: public, max-age=0, must-revalidate
💾 17,640 KiB saved on repeat visits ⚡ Instant repeat load 🔧 Effort: Low (15 min)
P1 — Critical

Google Fonts is render-blocking — delaying paint by ~1,740 ms

Lighthouse: Render-blocking requests — saves 1,740 ms

The Google Fonts stylesheet is loaded as a synchronous <link rel="stylesheet">, which blocks the browser from rendering anything until the font CSS downloads. Two font families with 8 total weights are loaded (Nunito: 400,500,600,700,800 + Manrope: 600,700,800).

✅ Fix 1 — Load fonts non-blocking with media swap trick
<!-- Replace the existing Google Fonts link with: --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link rel="preload" as="style" href="https://fonts.googleapis.com/css2?family=Nunito:wght@600;700;800&family=Manrope:wght@700;800&display=swap" onload="this.onload=null;this.rel='stylesheet'"> <noscript> <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Nunito:wght@600;700;800&family=Manrope:wght@700;800&display=swap"> </noscript>
✅ Fix 2 — Reduce font weights (you're loading 5 weights of Nunito, 3 of Manrope)
<!-- Trim to only what's actually used in CSS --> <!-- Nunito: only 600, 700, 800 are used (remove 400 and 500) --> <!-- Manrope: only 700 and 800 are used (remove 600) --> family=Nunito:wght@600;700;800&family=Manrope:wght@700;800
⏱ ~1,740 ms unblocked 📉 FCP improvement 🔧 Effort: Low (30 min)

P2 — High

All CSS is inlined per-page — no shared stylesheet caching

File audit: index.html is 75 KB; each of 16 HTML files has full inline CSS

Every page (index, service pages, blog, ICPs) duplicates the full CSS inline. A visitor navigating from the homepage to a service page re-downloads and re-parses the same ~1,200 lines of CSS. Extracting to a shared styles.css means the browser caches it after the first page load and reuses it across all 16 pages.

✅ Fix — Extract shared CSS to styles.css
<!-- In every HTML file, replace the <style> block with: --> <link rel="stylesheet" href="/styles.css"> <!-- Then set a long cache lifetime in _headers: --> /styles.css Cache-Control: public, max-age=31536000, immutable
💾 ~50 KB saved per page navigation ⚡ Instant CSS on page 2+ 🔧 Effort: Medium (2 hrs)
P3 — Medium

HubSpot tracking script adds a third-party network request

Lighthouse: 3rd parties flagged · File audit: hs-scripts.com loaded on every page

HubSpot is loaded with async defer (good — it doesn't block render), but it still adds a DNS lookup, connection, and download to a third-party domain on every page load. If HubSpot form tracking isn't actively being used, consider removing it or loading it only on the contact section.

✅ Fix — Delay HubSpot until user interaction
<!-- Replace the <script src="//js.hs-scripts.com/..."> with: --> <script> function loadHubSpot() { var s = document.createElement('script'); s.src = '//js.hs-scripts.com/46378159.js'; s.async = true; s.defer = true; document.head.appendChild(s); } // Load on first scroll or click — not on initial page paint window.addEventListener('scroll', loadHubSpot, { once: true }); window.addEventListener('click', loadHubSpot, { once: true }); </script>
📉 Faster initial paint 🔧 Effort: Low (20 min)
P3 — Medium

Legacy JavaScript detected (13 KB savings)

Lighthouse: Legacy JavaScript — saves 13 KiB

Lighthouse flagged legacy JS patterns — likely coming from the HubSpot or Google Analytics scripts (third-party). For any inline scripts in your HTML pages, make sure you're using modern ES6+ syntax (arrow functions, const/let, template literals) rather than transpiled ES5 code, which adds polyfill overhead.

💾 13 KB saved 🔧 Effort: Low
